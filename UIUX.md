# UI/UX Design Specification

## 1. 디자인 원칙
- **탐색 용이성:** 사용자가 원하는 시점의 데이터로 쉽게 이동할 수 있어야 한다.

## 2. 화면별 상세 설계

### 2.1. 메인 대시보드 (Main Dashboard)
* **Task Card UI:**
    * [아이콘 + 제목] --- [도넛/카운터]
    * [미니 히트맵 (최근 14일)]
    * *설명:* 메인에서는 공간 제약상 '지난달' 전체를 보여주지 않고, '최근 2주'만 보여주어 복잡도를 낮춤.

### 2.2. 상세 화면 (Task Detail) - *Updated*

* **Header Area:** (기존 동일)
* **Calendar Control (Month Navigator):**
    * 달력 그리드 바로 위에 컨트롤 배치.
    * **Layout:**
      ```
      [ < ]    2026년 1월    [ > ]
      ```
    * **Components:**
        * `<` (Chevron Left): 이전 달로 이동.
        * `Title`: 현재 보고 있는 년/월 (Bold Text).
        * `>` (Chevron Right): 다음 달로 이동 (미래 날짜 제한 없음 or 다음 달까지만 허용).
    * **Action:**
        * 버튼 탭 시 부드러운 전환 애니메이션과 함께 달력 데이터 갱신.
        * 현재 달이 아닌 경우, 우측 구석에 조그맣게 `[오늘]` 버튼 표시.

* **Calendar Grid:**
    * 월~일요일 헤더.
    * 날짜 셀(Cell): 데이터가 있는 날은 원형 컬러 마킹.
    * **Interaction:** 과거/현재/미래 상관없이 날짜 탭 -> **[수정 바텀 시트]** 호출.

### 2.3. 과거 수정 바텀 시트
* (기존 동일) 상세 화면 달력에서 날짜 선택 시 호출.

## 3. 시각적 스타일 가이드
* **Navigator:**
    * 화살표 아이콘은 터치 영역을 넉넉하게(44px 이상) 확보하여 오터치 방지.
    * 월 텍스트는 `H3` 정도의 크기로 명확하게 표시.
