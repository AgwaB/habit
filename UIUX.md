# UI/UX Design Specification

## 1. 디자인 원칙
- **공간 효율성:** 좁은 모바일 화면 내에서 정보 밀도를 높이되 복잡해 보이지 않게 한다.
- **직관적 흐름:** 왼쪽(과거)에서 오른쪽(오늘)으로 이어지는 시간의 흐름을 시각화한다.

## 2. 화면별 상세 설계

### 2.1. 메인 대시보드 (Main Dashboard) - *Finalized*

* **Task Card UI:**
    * 카드 하나가 **[오늘의 할 일 + 최근 2주간의 기록]**을 모두 담습니다.
    * **Layout:**
      ```
      [ Card Container (White, Shadow) ----------------------- ]
      |                                                        |
      |  [Row 1: 정보 및 액션]                                  |
      |  [Icon]  **독서하기** (Bold)             ( Donut )     |
      |          주 5회 목표                     ( Chart )     |
      |                                          1 / 5         |
      |                                                        |
      |  [Row 2: 흐름 (Mini Heatmap)]                          |
      |  □ ■ ■ □ ■ ■ □ ■ ■ ■ □ ■ ■ ■ (Today)                 |
      |  (지난 2주간의 기록: 왼쪽 -> 오른쪽)                      |
      |                                                        |
      [ ------------------------------------------------------ ]
      ```
    * **Mini Heatmap Detail:**
        * **모양:** 둥근 사각형 (Rounded Square) 또는 원형 (Circle).
        * **개수:** 화면 너비에 따라 7개(1주) ~ 14개(2주) 유동적 배치 (Mobile First).
        * **색상:**
            * 수행 O: `Primary Color` (진한 색)
            * 수행 X: `Light Gray` (연한 회색)
            * 오늘 날짜: 가장 오른쪽에 위치하며, 테두리(Border) 등으로 현재 위치 강조.

### 2.2. 상세 화면 (Task Detail)
* **Calendar:** 메인의 미니 히트맵으로 부족한 '전체 월간 뷰'를 제공.
* **History Edit:** 달력 날짜 탭 -> 바텀 시트 호출 -> 수정.

### 2.3. 과거 수정 (Bottom Sheet)
* 상세 화면 달력에서 진입. (메인에서는 불가)

## 3. 시각적 스타일 가이드
* **Card Spacing:** Row 1(제목/액션)과 Row 2(히트맵) 사이에 적절한 여백(Margin)을 두어 답답하지 않게 처리.
* **Touch Target:** 메인 화면의 미니 히트맵은 단순히 '보여주는 용도'이므로 터치 영역을 굳이 만들지 않거나, 터치 시 상세 화면으로 이동하도록 통일.
