# UI/UX Design Specification

## 1. 디자인 원칙 (Design Principles)
1.  **명확성(Clarity):** 모든 버튼과 숫자는 설명을 읽지 않아도 의미를 알 수 있어야 한다. 모호한 아이콘 대신 명확한 텍스트나 보편적 아이콘을 사용한다.
2.  **일관성(Consistency):** '완료'를 뜻하는 색상(예: Green/Brand Color)은 어디서든 동일한 의미로 쓰인다. 카드 UI의 여백과 배치는 모든 리스트에서 동일하다.
3.  **시각적 계층(Hierarchy):** 사용자가 가장 먼저 봐야 할 것은 '오늘 했는가?(체크버튼)'와 '얼마나 했는가?(도넛차트)'이다. 수정/삭제 등의 관리 기능은 숨기거나 작게 배치한다.

## 2. 화면별 상세 설계 (Screen Details)

### 2.1. 메인 대시보드 (Main Dashboard)
앱의 홈 화면. 사용자는 이 화면에서 90%의 시간을 보낸다.

* **Header:**
    * 좌측: 오늘 날짜 (예: 1월 18일 월요일)
    * 우측: 설정 아이콘 (톱니바퀴)
* **Task List (Card UI):**
    * 각 Task는 카드 형태의 컨테이너를 가짐.
    * **배치 구조:**
        ```
        [ 좌측 영역: 정보 ]                [ 우측 영역: 액션 & 피드백 ]
        ---------------------------------------------------------
        [아이콘]  Task 제목              ( 도넛 차트 ⭕️ )
                  "주 3회 목표"          가운데 숫자: 1/3
        ```
    * **도넛 차트 인터랙션 (핵심):**
        * **Goal Mode:** 얇은 회색 링 위에, 브랜드 컬러가 진행률만큼 채워짐.
        * **Counter Mode:** 링 없이 굵은 숫자로 누적 횟수만 표시.
        * **클릭 영역:** 카드 전체를 누르면 [상세 화면] 이동. 우측 도넛/숫자 영역을 누르면 [카운트 증가].
* **Floating Action Button (FAB):**
    * 화면 우측 하단 `+` 버튼. 새로운 습관 추가.

### 2.2. 습관 추가/수정 (Add/Edit Modal)
사용자의 입력을 최소화하는 폼.

* **입력 필드:**
    1.  **제목:** "어떤 습관인가요?" (Placeholder: 운동하기)
    2.  **유형 선택 (Segmented Control):** [ 목표 설정 ] vs [ 단순 기록 ]
    3.  **빈도 설정 (목표 설정 선택 시 등장):**
        * 주기: [매일 / 주간 / 월간] 드롭다운
        * 횟수: `+ / -` 스텝퍼 버튼 (Default: 1)
* **저장 버튼:** 화면 하단에 꽉 찬 버튼 (Primary Color). "시작하기" 문구 사용.

### 2.3. 상세 화면 (Task Detail)
특정 습관의 깊이 있는 데이터를 확인하는 곳.

* **Header:**
    * `<` (뒤로가기), 제목, `...` (수정/아카이브 메뉴)
* **Summary Section:**
    * 가장 눈에 띄게 큰 텍스트로 **"이번 주 달성률 66%"** 표시.
    * 바로 아래 **Current Streak 🔥 5일** 표시.
* **Calendar Section (Task Specific):**
    * 월간 달력 뷰.
    * 수행한 날짜에는 브랜드 컬러의 원형 배경(●)이 깔림.
    * 수행하지 않은 날은 숫자만 표시.
* **History Section (Heatmap):**
    * "지난 활동 기록" 타이틀.
    * 가로 스크롤 가능한 5행(요일)짜리 히트맵 그리드. (좌측이 과거 -> 우측이 오늘)

## 3. 시각적 스타일 가이드 (Visual Style)

* **Color Palette:**
    * **Primary:** 시선을 끄는 메인 컬러 (예: Indigo 또는 Teal). 체크 완료 및 도넛 차트 게이지에 사용.
    * **Background:** Off-white (#F9F9F9)로 눈의 피로 감소.
    * **Card BG:** Pure White (#FFFFFF) + 은은한 그림자(Shadow)로 계층 구분.
    * **Text:** #333333 (본문), #888888 (보조 설명).
* **Typography:**
    * 숫자(카운트, 날짜)는 가독성이 좋은 `Monospace` 혹은 `San-serif` 볼드체 사용.
* **Feedback:**
    * 체크 완료 시 미세한 햅틱(진동) 혹은 도넛 차트가 튀어오르는 애니메이션 효과 적용.

## 4. 예외 케이스 처리 (Exception Handling)
* **데이터 없음:** 히트맵이나 달력에 데이터가 없을 경우 "아직 기록이 없습니다. 오늘부터 시작해보세요!" 문구 표시.
* **네트워크 오류:** "오프라인 상태입니다" 토스트 메시지 표시 (로컬 낙관적 업데이트 후 추후 동기화).
