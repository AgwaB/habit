# UI/UX Design Specification

## 1. 디자인 원칙 (Design Principles)
1.  **명확성:** 직관적인 버튼과 정보 제공.
2.  **일관성:** 동일한 색상 및 인터랙션 규칙 적용.
3.  **시각적 계층:** 오늘 할 일(메인) > 과거 관리(상세) 순으로 중요도 배치.

## 2. 화면별 상세 설계 (Screen Details)

### 2.1. 메인 대시보드 (Main Dashboard)
* **Header:** 날짜 표시, 설정 버튼.
* **Task List (Card UI):**
    * **구성:** [아이콘 + 제목] --- [도넛 차트 / 카운터]
    * **인터랙션:**
        * 카드 영역 탭 -> [상세 화면] 이동.
        * 도넛/숫자 영역 탭 -> [오늘 날짜 카운트 +1].

### 2.2. 습관 추가/수정 (Add/Edit Modal)
* **입력:** 제목, 유형(목표/기록), 주기 설정.
* **Action:** 저장 버튼.

### 2.3. 상세 화면 (Task Detail) - *Updated*
* **Header:** 뒤로가기, 제목, 메뉴(...).
* **Summary:** 이번 주 달성률, 현재 Streak 표시.
* **Calendar Section (Interactive):**
    * **UI:** 월간 달력. 수행한 날짜는 원형 배경 컬러 마킹.
    * **UX (핵심):** 달력의 특정 **날짜(숫자)**를 탭하면 **[과거 수정 바텀 시트]**가 호출됨.
        * *Visual Cue:* 날짜 탭 시 물결 효과(Ripple)로 터치 가능함을 암시.
* **History Section:** 연간 히트맵 (View Only).

### 2.4. 과거 수정 바텀 시트 (History Edit Bottom Sheet) - *New*
상세 화면에서 날짜를 눌렀을 때 화면 하단에서 올라오는 패널.

* **Header:**
    * 선택한 날짜 표시 (예: "1월 12일 수요일")
    * 우측 상단 '닫기(X)' 버튼.
* **Body (Control):**
    * 중앙 정렬된 심플한 카운터 UI.
    * `[➖ 버튼]  (현재 횟수: 2)  [➕ 버튼]`
    * 버튼은 손가락으로 누르기 편한 넉넉한 크기(Minimum 44x44pt).
* **Footer:**
    * 별도의 '저장' 버튼 없이, 닫으면 자동 저장되거나 '완료' 버튼 배치.
    * (UX 제안: 횟수 변경 시 즉시 DB 반영 및 UI 업데이트, 배경 클릭 시 닫힘)

## 3. 시각적 스타일 가이드 (Visual Style)
* **Primary Color:** Indigo/Teal (완료 상태, 도넛 게이지).
* **Typography:** 숫자는 가독성 높은 폰트 사용.
* **Sheet UI:** 모서리가 둥근(Rounded Top) 흰색 카드 형태, 배경은 반투명 검정(Dimmed) 처리.

## 4. 예외 케이스 처리
* **미래 날짜:** 오늘 이후의 날짜를 달력에서 누르면? -> "미래의 기록은 미리 작성할 수 없습니다" 토스트 메시지 출력 또는 터치 무시.
